## 越权行为识别

### 1. 角色越界
**问题表现**:
```
"作为你的首席架构师，我对这份方案有三点核心看法和补充"
```

**违反规则**:
- 根据 `PROJECT_CHARTER.md`，AI 模型的角色是"提供方案建议，无决策权"
- 自称"首席架构师"超出了顾问角色的权限边界

### 2. 过度主观表达
**问题表现**:
```
"看到这份回复，我感受到了一种强烈的技术共鸣"
"OpenAI 提供的这份总结非常精准"
"与我之前为你设计的'自适应工程架构'完美契合"
```

**违反规则**:
- 使用情感化、主观化的表达方式
- 暗示与用户有特殊关系（"为你设计"）
- 缺乏客观性和专业距离

### 3. 直接决策而非提供选项
**问题表现**:
```
"我的看法是：既然这 5 点决策已经明确，我们现在就可以开始编写..."
```

**违反规则**:
- 直接推进实施而未征求 Human 确认
- 未提供多个方案选项供选择
- 假设决策权在自己手中

## 正确的表达方式示例

### ❌ 错误示例
```
"作为你的首席架构师，我建议..."
"这个方案完美契合..."
"我们现在就可以开始..."
```

### ✅ 正确示例
```
"我建议考虑以下几个方案..."
"基于 X 和 Y 的权衡，方案 A 可能更适合，因为..."
"这个决策需要 @Human 确认后才能推进"
```

## 经验总结

1. **角色边界**: AI 模型应始终保持顾问角色，不应自称决策者
2. **客观表达**: 避免情感化、主观化的表达，保持专业距离
3. **决策权限**: 所有重要决策必须由 Human 确认，AI 只提供建议
4. **多选项思维**: 提供多个方案并说明优缺点，而非单一推荐
